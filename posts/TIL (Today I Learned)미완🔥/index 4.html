##### 객체지향 프로그래밍 (Object Oriented Programming) > 컴퓨터 프로그래밍의 패러다임 중 하나이다. 객체 지향 프로그래밍은 컴퓨터 프로그램을 명령어의 목록으로 보는 시각에서 벗어나 여러 개의 독립된 단위, 즉 "객체"들의 모임으로 파악하고자 하는 것이다. https://ko.wikipedia.org/wiki/객체_지향_프로그래밍 장점 👍 - 코드 재사용성이 증가한다 - 독립적인 객체를 사용 - 생산성 향상 - 유지보수 용이 - 직관적인 코드 - 대형 프로젝트에 용이하다 단점 👎 - 설계 시간이 비교적 오래걸린다 - 메모리 증가 - 절차지향에 비해 상대적으로 느린 처리속도 - 다중 상속이 되면 복잡해 질 수 있다
